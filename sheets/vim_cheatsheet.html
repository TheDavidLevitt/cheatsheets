Vim Survival Cheat Sheet

Get out of it, then use it · started 2026-06-05 · twin of Tools/vim_cheatsheet.md

0 · First, how to GET OUT

Trapped? Press Esc a couple of times (back to Normal mode), then type :q! and Enter — quits and discards everything. That one sequence rescues 90% of "stuck in Vim" moments.
:wq Entersave and quit (or ZZ, or :x)
:w Entersave (write), stay open
:q Enterquit (only if no unsaved changes)
:q! Enterquit, discard all edits (! = force)
:qa! Enterquit ALL windows, discard (qa = quit all) — for vimdiff / multiple splits, where :q! only closes one
Vim = "Vi IMproved", a modal text editor (Vi: Bill Joy, 1976 → Vim: Bram Moolenaar, 1991). Modal = the same key does different things per mode — alien at first, fast once it clicks. You keep landing in it because tools open $EDITOR (default Vim on most systems): git commit, git config --global --edit, git rebase -i, crontab -e.

1 · The three modes (core mental model)

            i / a / o ...
   ┌────────────────────────►┐
NORMAL                     INSERT   ← type text normally here
(navigate,                   │
 run commands)  ◄────────────┘
   │   ▲          Esc
 : │   │ Esc
   ▼   │
COMMAND-LINE     VISUAL  ← (v) select text to act on
(:w :q :s ...)
Normal (default) — keys are commands, not text (why fresh Vim looks "erratic" — you're firing commands). Insert — types text; enter with i, leave with Esc. Visual — highlight a range then act. Command-line — the : prompt.
Vim convention — the mode indicator: look at the bottom-left of the terminal. -- INSERT -- = Insert mode; blank = Normal mode. Glance there before typing.

2 · Entering Insert mode (several doors in)

Pick the right entry

iinsert before cursor
aappend after cursor
Iinsert at line start (first non-blank)
Aappend at line end
oopen new line below + type
Oopen new line above

Esc always returns to Normal mode.

The Git "uncomment & save" flow

In Git config, comments start with ; or #; "uncomment" = delete that leading char.

1. Move cursor onto the ; or #   (arrows or h j k l)
2. Press  x   ← deletes that one char (no Insert needed!)
   or:  i , then Backspace it
3. Need a value? press  a , type it
4. Esc   (-- INSERT -- disappears)
5. :wq  Enter   to save & quit
   messed up? Esc , :q!  Enter
Pro tip: in Normal mode, x deletes the character under the cursor instantly — often quicker than entering Insert mode just to Backspace one ;.

3 · Moving around (Normal mode)

Motions

h j k lleft, down, up, right
w / bforward / back a word
0 / $start / end of line
gg / Gtop / bottom of file
:42 Enterjump to line 42
Ctrl-d / Ctrl-uhalf-page down / up

Why hjkl?

On Bill Joy's 1970s ADM-3A terminal those four keys had arrow symbols printed on them — Vi inherited the layout. j looks a bit like a down-arrow → "j = down". Arrow keys also work; home-row keys just keep your hands put.

4 · Editing without Insert mode (Normal-mode power)

Speed = operator + motion. d delete · c change (delete + Insert) · y yank (copy). Read it as verb + count + noun: d2w = delete 2 words, 3dd = delete 3 lines, ci" = change inside quotes.

Delete / change / copy

xdelete char under cursor
dddelete (cut) whole line
dw / Ddelete to end of word / line
ccchange whole line (clears + Insert)
yy / pyank line / paste after cursor
r<char>replace one char, no Insert

Undo & repeat

uundo
Ctrl-rredo
.repeat last change (hugely useful)

5 · Search & replace

/text Entersearch forward; n next, N previous
?text Entersearch backward
:%s/old/new/greplace old→new everywhere (% = whole file, g = all per line)
:%s/old/new/gcsame, confirm each
:noh Enterclear search highlight

Grammar: :[range]s/pattern/replacement/[flags] — s = substitute, inherited from ed/sed. Same s/old/new/ you see in shell one-liners.

STUCK?            Esc Esc  then  :q!  Enter      (quit, discard)
SAVE & QUIT       Esc  :wq  Enter
TYPE TEXT         i   ... (then Esc when done)
DELETE A CHAR     x
DELETE A LINE     dd
UNDO / REDO       u  /  Ctrl-r
FIND              /word  Enter   (n = next)

7 · Bonus — make this stop happening

Prefer a friendlier editor for Git? 
git config --global core.editor "nano"          # simple terminal editor
git config --global core.editor "code --wait"   # VS Code (waits till you close the tab)
--wait matters: without it VS Code returns instantly and Git sees an empty message. To learn Vim properly, run vimtutor — the best 30 minutes you'll spend.
