Obsidian Cheat Sheet

Local-first notes: vault · links · plugins · started 2026-06-06 · twin of Tools/obsidian_cheatsheet.md

0 · The mental model (read once)

Obsidian = free note app (2020, by the makers of Dynalist): your notes are plain Markdown files in an ordinary folder on your disk — no database, no account, no lock-in. Name = geology pun: obsidian is volcanic glass, lava cooled fast into something hard and sharp. Markdown = Gruber's 2004 plain-text formatting syntax (# heading, **bold**) — "markdown" jokes on "markup". Local-first = your disk is the source of truth; cloud is an optional copy (opposite of Notion).
Two layers — the single most important idea: 
layer 2:  the GRAPH   ← links between notes (Obsidian computes this)
              ▲
layer 1:  the VAULT   ← a plain folder of .md files (you own this)
Vault = the folder Obsidian opens; every note is a real .md file, subfolders are just subfolders. Delete the app and your notes still open in any editor. Graph = the link network Obsidian builds on top — links turn a pile of files into a wiki. Only app-private data is the hidden .obsidian/ config folder (analogous to Git's .git/) — your content never lives there.
Personal-wiki fit: one growing, interlinked, plain-text knowledge base is exactly what Obsidian is shaped for — and the vault can live in a Git repo (§7).

1 · Markdown syntax Obsidian uses

Standard Markdown plus a few Obsidian extensions (⊕).

Standard

# H1 … ###### H6headings (count of # = level)
**bold** / *italic*bold / italic
- item / 1. itembullet / numbered list (indent to nest)
- [ ] taskcheckbox — click toggles to - [x]
`code` / ``` fenceinline code / code block (add a language for highlighting)
> quoteblockquote
[text](https://url)standard Markdown link
![alt](image.png)embed an image

Obsidian extras ⊕

[[Note name]]wikilink — see §2
![[Note name]]embed (transclude) another note inline
==highlight==highlighted text
%%hidden%%comment — invisible in reading view
---horizontal rule; at the very top it fences frontmatter — a YAML metadata block (tags, dates, custom fields)

YAML = "YAML Ain't Markup Language" (recursive-acronym joke); human-readable key: value data format.

2 · Links: wikilinks, backlinks, graph

Linking is the core skill — each note a node, each link an edge.

Link syntaxes

[[Attention]]wikilink (from wiki software, WikiWikiWeb 1995 → Wikipedia). Type [[ → autocomplete. The default, low-friction choice
[[Attention|see this]]alias — | sets display text
[[Attention#History]]link to a heading inside a note
[Attention](Attention.md)markdown link — more portable, clumsier to type. Toggle in Settings → Files & links

Backlinks & graph

Linking to a nonexistent note is fine — link shows dimmed; clicking creates it. "Link first, write later" is how the wiki grows.

Backlinks = computed for free: every note's "Linked mentions" pane lists all notes pointing to it. You only write links forward.

Graph view (⌘G / Ctrl+G) = whole vault as a network. Good for spotting orphans & clusters; the backlinks pane does the daily work.

3 · Tags vs. links vs. folders

Links [[...]]specific note-to-note relationships — the primary tool, default to these
Tags #topiccategories & status cutting across many notes (#unread, #gcp/storage — nesting works); click a tag to see everything carrying it
Folderscoarse, mutually-exclusive buckets only (daily/, templates/); a note lives in one folder but can carry many links/tags — weakest tool
Community rule of thumb: link generously, tag sparingly, folder reluctantly. Deep folder trees recreate the rigid hierarchy the link graph was supposed to free you from.

4 · Essential hotkeys (Mac · Windows)

The big three

⌘P · Ctrl+PCommand palette — fuzzy-search every command by name (like VS Code's ⌘⇧P); finds any hotkey you forgot
⌘O · Ctrl+OQuick switcher — fuzzy-jump to any note (O = open); type a new name + Enter to create it
⌘⇧F · Ctrl+Shift+FSearch vault — full-text, with tag: path: file: operators

Next tier

⌘F · Ctrl+Fsearch current note
⌘N · Ctrl+Nnew note
⌘E · Ctrl+Etoggle edit ↔ reading view
⌘G · Ctrl+Ggraph view
⌘Click · Ctrl+Clickfollow link under cursor
⌘, · Ctrl+,settings (all hotkeys rebindable: Settings → Hotkeys)
Vim-user note: built-in Vim mode — Settings → Editor → "Vim key bindings". Modal editing (hjkl, operator+motion, / search) inside the note editor. A solid subset, not full Vim — no .vimrc (community plugin "Vimrc Support" adds one). See vim_cheatsheet.html.

5 · Daily notes & templates

Daily notes

One auto-named note per day (default YYYY-MM-DD.md — already ISO). The answer to "where do I write this right now?"

EnableSettings → Core plugins → Daily notes
Open todaycalendar ribbon icon, or bind a hotkey to "Open today's daily note"
Configuredate format, target folder (e.g. daily/), template

Pattern: daily note = inbox (chronological, disposable); wiki notes = library (topical, permanent). Links connect the two.

Templates

A stub note inserted into the current note on demand — skeleton headings, frontmatter, boilerplate.

EnableSettings → Core plugins → Templates; point at a folder (e.g. templates/)
Insertcommand palette → "Templates: Insert template"
Placeholders{{title}}, {{date}}, {{time}}
Daily-note templateset in Daily notes settings — every day starts with the same skeleton

Need logic (conditionals, prompts, JS)? → Templater community plugin (§6).

6 · Community plugins

Core plugins = first-party toggles. Community plugins = third-party extensions from the in-app marketplace, written in JS/TS against Obsidian's public API (Application Programming Interface — the functions an app exposes for other code to call).

Enable: Settings → Community plugins → turn off Restricted mode → Browse → install → enable. Restricted mode defaults on because plugins run arbitrary code — same trust call as a VS Code extension. Stick to popular, maintained ones.

Four widely-recommended starters

Dataviewquery the vault like a database — list notes by tag/date, build tables from frontmatter fields
Templatertemplates with logic: variables, prompts, conditionals, JS — superset of core Templates
Calendarmonth-view sidebar; click a day → that daily note
Excalidrawhand-drawn-style sketches/diagrams embedded as notes
Start with zero plugins; add one when you feel a specific pain. Plugin sprawl = the classic failure mode (fiddling with the system instead of writing notes).

7 · Sync options compared

A vault is just a folder, so anything that syncs folders can sync it — with trade-offs.

Obsidian Sync (paid)first-party; E2EE (end-to-end encrypted — only your devices hold the key), version history, all platforms incl. mobile. The "it just works" choice
iCloud Drive (free)vault inside the Obsidian iCloud folder — official free path for Mac ↔ iPhone/iPad. Apple-only; occasional lag / conflict files; no Android
Git (free)vault = repo; push to GitHub manually or via the Obsidian Git plugin. Full history + offsite backup, familiar tooling; clunky on mobile; don't push private notes to a public repo
Dropbox / Drive / Syncthinggeneric folder sync — fine on desktop, poor-to-hacky on mobile
One rule regardless of method: never let two sync systems manage the same vault (e.g. iCloud + Git auto-commit) — that's the recipe for conflict files. Pick one transport; Git can sit underneath as a deliberate, manual backup layer.

8 · Teardown — closing, removing, walking away

Close / switch vaultvault-switcher icon (bottom-left) → open another vault or "Manage vaults"; each vault is its own window
Remove vault from launcherManage vaults → ✕ — only forgets the bookmark; folder and notes stay on disk
Delete a vault for realdelete the folder in Finder/Explorer — a vault is just a folder, so this deletes the notes. Flip side of local-first: nothing survives in a cloud account. Back up first
Remove a community pluginSettings → Community plugins → ✕ Uninstall ("Disable" is the reversible half-step; uninstall deletes its code from .obsidian/plugins/)
Disable sync safelyconfirm all devices are fully synced first, then turn off the transport (Obsidian Sync / move vault out of iCloud / stop the Git plugin). Notes are local-first so nothing is deleted — but un-synced edits elsewhere become a fork
