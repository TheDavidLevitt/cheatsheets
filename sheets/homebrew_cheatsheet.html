Homebrew Cheat Sheet

macOS package manager: install · upgrade · maintain · started 2026-06-06 · twin of Tools/homebrew_cheatsheet.md

0 · The mental model (read once)

Homebrew = the open-source package manager for macOS (Max Howell, 2009). A package manager installs, upgrades, and removes other software for you — resolving dependencies and tracking versions. Linux ships with one (apt, dnf…); macOS doesn't, so the community built Homebrew. The name = "home-brewed" software, and the whole vocabulary extends the beer-brewing pun.
A brewery for software — decode the metaphor once, the commands explain themselves: 
tap          →   formula / cask   →   bottle or build   →   keg, in the Cellar
(a repo of       (a recipe for         (prebuilt binary       (the versioned
 recipes)         one package)          or built from src)      installed copy)
brewthe command itself — "brew up" some software
formula (pl. formulae)a brewer's recipe → script describing how to download + build one CLI package
caskbarrel of finished drink → recipe for a prebuilt GUI app (Chrome, VS Code…) — arrives finished, no building
Cellarwhere barrels are stored → the install directory (/opt/homebrew/Cellar on Apple Silicon)
kegone barrel → one installed version of one formula, e.g. Cellar/python/3.13.1/
taptap a keg to pour from it → add a third-party repo of recipes
bottlebeer bottled for distribution → prebuilt binary; brew "pours" it instead of compiling — why installs take seconds
Command grammar: brew <verb> [options] [package] — same Unix flag convention as Git (short -n, long --cask). Manual: brew help <verb>. Why Cellar + symlinks? Each keg lives in its own versioned folder; executables are symlinked (filesystem shortcuts) into /opt/homebrew/bin. Clean upgrades, easy rollbacks.

1 · Installing Homebrew itself

One command from brew.sh — fetches and runs the installer via curl (the standard CLI URL-fetcher). It asks for your password once, then tells you how to add brew to PATH — read that final output.

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Apple Silicon note (M1–M4)

On Apple Silicon, Homebrew lives in /opt/homebrew. Older Intel Macs used /usr/local — translate when old tutorials hard-code it. Check yours: brew --prefix.

PATH — how the shell finds brew

PATH = environment variable listing directories the shell searches for commands. The installer appends this to ~/.zprofile:

eval "$(/opt/homebrew/bin/brew shellenv)"

"command not found"? That line is missing. Debug with echo $PATH and which brew.

2 · Everyday verbs

Find & inspect

brew search pythonsearch formulae + casks by name
brew info pythondescription, version, deps, install state
brew listeverything installed (--cask for casks)
brew outdatedinstalled packages with newer versions available
brew deps --tree pythondependency tree of a formula

Install & remove

brew install pythonpour a bottle into the Cellar + symlink
brew uninstall pythonremove the keg and its symlinks
brew list --versions pythonwhich version(s) of one formula you have
brew pin pythonprotect from blanket brew upgrade (unpin releases)

3 · update vs upgrade — the classic confusion

Not synonyms. brew update refreshes Homebrew's own metadata — git-pulls the taps so brew knows the latest versions. Installs nothing. ("update the catalog") · brew upgrade actually installs the newer versions of your packages. ("upgrade the goods")

The ritual

brew updaterefresh recipe books — learn what's new
brew upgradeinstall newer versions of everything outdated
brew upgrade pythonupgrade just one package
brew update && brew upgradetypical maintenance one-liner

Modern brew auto-runs update before installs — but the distinction saves you from "why didn't update update my packages?"

Casks — GUI apps via CLI

A cask installs a regular macOS GUI app into /Applications — no website, no .dmg, no dragging.

brew install --cask firefoxinstall a GUI app
brew uninstall --cask firefoxremove it
brew upgrade --caskupgrade all version-reporting casks
brew list --casklist installed casks

4 · Maintenance — doctor, cleanup, autoremove

Health & disk space

brew doctordiagnose problems (broken symlinks, PATH, stale taps) — "ready to brew" = all clear
brew cleanupdelete superseded kegs + stale download cache
brew cleanup -ndry run (-n): show what would go, delete nothing
brew autoremoveremove orphaned dependencies no longer needed
brew --prefixprint install root (/opt/homebrew on Apple Silicon)

brew services — background daemons

Daemon = a long-running background program (e.g. a database server). macOS manages them via launchd; brew services is a friendly wrapper.

brew services listall brew services + status
brew services start postgresqlstart now and auto-start at login
brew services run postgresqlstart now only — no auto-start
brew services stop postgresqlstop + unregister from login
brew services restart postgresqlstop + start (after config change)
Full teardown — beyond uninstall/cleanup/autoremove: brew uninstall --zap firefox removes a cask plus its app data (prefs, caches, support files — the cask's zap stanza; regular uninstall leaves those behind) · brew untap user/tap removes a third-party tap (§5) · to remove Homebrew itself, run the official uninstall script — mirror of the install one-liner (§1): /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/uninstall.sh)" (it lists any leftover files for manual deletion).

5 · Taps — third-party recipe repositories

Default taps: homebrew/core (formulae) and homebrew/cask (casks). A tap adds a third-party GitHub repo of extra recipes.

Tap commands

brew taplist taps currently added
brew tap hashicorp/tapadd a tap — shorthand for github.com/hashicorp/homebrew-tap
brew install hashicorp/tap/terraforminstall from a specific tap (user/tap/formula)
brew untap hashicorp/tapremove a tap

⚠ Trust

Taps are just git repos of formula scripts — and a formula runs code on your machine. Only tap sources you'd trust as much as any software vendor.
