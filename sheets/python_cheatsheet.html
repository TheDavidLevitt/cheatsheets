Python = high-level, interpreted, dynamically-typed language (Guido van Rossum, 1991; named after Monty Python, not the snake). Interpreted = executed line-by-line, no compile step you manage. Dynamically typed = types checked at runtime; a name can be re-bound to any type. Mental model: everything is an object; every variable is a name bound to an object — a label on a box, not the box. Most gotchas (§8) fall out of that.

1 · Syntax essentials

indentation · f-strings · if · loops · comprehensions · slicing
Indentation IS the syntax. Blocks are marked by leading whitespace, not braces; a colon : announces "indented block follows." Standard = 4 spaces (PEP 8). Other grammar: indexing starts at 0 · underscores in numbers ignored (1_000_000) · case-sensitive names · no semicolons · round() uses banker's rounding (ties → even digit: round(2.5) → 2).

Comments & docstrings

# comment — stripped at parse time

def f():
    """Docstring: 1st statement of a
    function/class/module. Lives at
    runtime as f.__doc__ (help, IDEs)."""

f-strings (3.6+) — f"..." makes {} an expression slot

f"Hello {name}"any expression inside {}
f"{x=}"debug form → x=42 (3.8+)
f"{3.14159:.2f}"format spec after colon → 3.14
f"{1234567:,}" / f"{0.85:.1%}"thousands sep · percent
f"{{literal}}"double braces = literal braces
r"raw\path" · b"bytes"raw string (regex, paths) · bytes

Conditionals

if score >= 90:   grade = "A"
elif score >= 80: grade = "B"   # else-if
else:             grade = "C"

# ternary — inline if that RETURNS a value:
status = "adult" if age >= 18 else "minor"

Operators: == != < <= > >= + the words and or not. Chaining works: 0 <= x < 10. Truthiness: falsy = False None 0 0.0 "" [] {} set(); so if my_list: = "if non-empty".

Loops

for item in items: ...          # any iterable
for i in range(5): ...          # 0..4 (stop excluded)
for i, x in enumerate(items): . # index + value
for a, b in zip(xs, ys): ...    # lockstep pairs
while balance > 0: ...          # repeat while true

break = exit loop · continue = next iteration · pass = placeholder no-op. zip stops at the shortest input — strict=True (3.10+) to raise instead.

Comprehensions — build a collection in one line

[x**2 for x in range(10)]list
[x for x in nums if x % 2 == 0]list + filter (the if is optional)
{k: len(k) for k in words}dict
{x % 3 for x in nums}set (uniques)
(x**2 for x in nums)generator — lazy, computed on demand

Read: "a list of expr for each item in iterable where cond holds."

Slicing — seq[start:stop:step]

start included, stop excluded — the #1 surprise. Negative = from the end. On s = "Python":

s[0] / s[-1]'P' / 'n' (first / last)
s[0:2]'Py' — indices 0,1
s[:3] / s[3:]'Pyt' / 'hon'
s[::2] / s[::-1]every 2nd · reversed
lst[:]shallow copy idiom

3 · Functions

def · defaults · *args/**kwargs · lambda · LEGB · decorators
Function vs method: a function (len(obj)) takes the object into it; a method (obj.append(x)) is called on the object and belongs to its class.

def · return · defaults

def power(base, exp=2):        # default arg
    """Return base ** exp."""
    return base ** exp          # no return → None

power(3)              # 9
power(exp=3, base=2)  # 8 — keyword args, any order

*args / **kwargs — collect & unpack

def f(*args, **kwargs)collect: extras → tuple / dict
f(*lst) / f(**d)unpack: spread into positional / keyword args
power(**{"base":2,"exp":5})→ 32 — dict keys become kwargs (tool-calling bridge)

Mnemonic: dict = two infos per entry (key+value) → double asterisk. Full story: python_wiki.md § Argument unpacking.

lambda — anonymous one-expression function

add_one = lambda x: x + 1
sorted(words, key=lambda w: len(w))   # the real use

Etymology: Alonzo Church's lambda calculus (1930s formal logic of anonymous functions). Needs a name or a 2nd line? Use def.

Scope — LEGB & decorators

LEGB = name lookup order, innermost first: Local → Enclosing → Global → Built-in. Assignment in a function makes a local name unless global/nonlocal (usually a smell — pass in / return out).

Decorator = function that wraps a function; @timer above def slow(): is sugar for slow = timer(slow). Read @app.route(...), @server.tool() as "register the function below."

Workhorse built-ins

len · sorted(it, key=, reverse=)count · new sorted list
min · max · sumaggregates
any(it) / all(it)True if any / every element truthy
map(f, it) / filter(f, it)lazy transform/filter (comprehensions usually read better)
next(iterator, default)pull one — "first match" idiom
help(obj) / dir(obj)docs / list attributes

4 · Classes (brief)

blueprint · self · __init__ · dunders · inheritance
Class = blueprint for creating objects; instance = one object built from it. Attributes = data on the object; methods = functions in the class — their first parameter self is the instance itself.

Anatomy

class Dog:
    species = "Canis familiaris"   # class attr (shared)

    def __init__(self, name, age): # constructor
        self.name = name           # instance attrs
        self.age = age

    def bark(self):
        return f"{self.name} says woof"

rex = Dog("Rex", 3)    # instantiation → __init__
rex.bark()

class Puppy(Dog):      # inheritance
    def bark(self): return "yip!"   # override

Dunder methods — hooks Python calls FOR you

"Dunder" = double underscore. Implementing them makes a class behave like a native type ("Pythonic").

__init__constructor — runs at ClassName(...)
__str__readable string — print(x), str(x)
__repr__unambiguous dev string — REPL echo
__len__ · __eq__ · __getitem__len(x) · x == y · x[key]
__enter__ / __exit__context-manager protocol (§6 with)

Data-carrying classes: @dataclass (stdlib, auto __init__/__repr__/__eq__) or Pydantic (adds runtime validation).

5 · Errors & exceptions

try/except/else/finally · raise · who's-who table
Exception = object raised when something goes wrong; unhandled → crash with a traceback. Read tracebacks bottom-up: last line = the error, lines above = how it got there.

The full shape

try:
    value = int(user_input)
except ValueError:               # specific type
    print("not a number")
except (KeyError, IndexError) as e:
    print(f"lookup failed: {e}")
else:
    ...        # only if NO exception
finally:
    cleanup()  # ALWAYS runs

raise ValueError("bad input")    # raise your own

Catch the narrowest type you can handle; bare except: hides bugs. Don't use exceptions where an if will do.

Who raises what

ValueErrorright type, bad value — int("abc")
TypeErrorwrong type — len(42), unexpected kwarg
KeyError / IndexErrormissing dict key (use .get) / index out of range
AttributeErrorno such attribute — some_list.tool_use
NameErrorname never defined (often a typo)
FileNotFoundErroropen() on a missing path
ZeroDivisionErrorx / 0
ModuleNotFoundErrorpackage not installed in this environment
KeyboardInterruptyou pressed Ctrl-C

6 · Files & the with-statement

open modes · context managers · JSON in two lines

The pattern

with open("notes.txt") as f:   # bind + GUARANTEE close
    text = f.read()             # whole file
# closed here, even on exception

with open("notes.txt") as f:
    for line in f:              # files iterate by line
        process(line.rstrip())

with open("out.txt", "w") as f:
    f.write("hello\n")

with works on any context manager (__enter__/__exit__) — files, locks, DB connections, SDK clients. Why closing matters (file descriptors, buffers): python_wiki.md § File I/O.

open() modes & JSON

"r"read text (default)
"w"write text, truncates existing
"a"append text
"rb" / "wb"binary (images, pickle, parquet)
with open("data.json") as f:
    data = json.load(f)
with open("out.json", "w") as f:
    json.dump(data, f, indent=2)

Trailing-s mnemonic: loads/dumps = string, not plural. JSON↔Python mapping table: python_wiki.md.

8 · Gotchas — the classic traps

aliasing · mutable defaults · is vs == · // and floats

Copy vs reference (aliasing)

a = [1, 2, 3]
b = a              # ALIAS — same object
b.append(4)
a                  # [1, 2, 3, 4]  ← a changed too!

b = a.copy()       # shallow copy (a[:], list(a))
b = copy.deepcopy(a)  # also copies nested objects

Assignment never copies — it binds another name to the same object. Shallow = new outer container, nested objects still shared; deep = recurses. (Objects are freed when their reference count hits zero — garbage collection is bookkeeping of exactly these aliases.)

Mutable default arguments

def add_item(item, bucket=[]):   # BUG
    bucket.append(item); return bucket
add_item(1)   # [1]
add_item(2)   # [1, 2]  ← shared list!

def add_item(item, bucket=None): # FIX: None sentinel
    if bucket is None:
        bucket = []

Defaults are evaluated once, at def time — a mutable default is shared across every call.

is vs ==

[1,2] == [1,2]   # True  — equal VALUES (__eq__)
[1,2] is [1,2]   # False — different objects
x is None        # the ONE place to use `is`

Small ints/short strings are cached so a=5; b=5; a is b happens to be True — never rely on it.

Division & float surprises

7 / 2    # 3.5  — / ALWAYS float (Py3)
7 // 2   # 3    — floor division (−7//2 == −4)
7 % 2    # 1    — modulo
0.1 + 0.2 == 0.3            # False!
math.isclose(0.1+0.2, 0.3)  # True — use tolerance
round(2.5)                  # 2 — banker's rounding
Quick-fire: modifying a list while looping over it → loop a copy for x in lst[:]: · x = lst.sort() loses data (returns None) — use sorted(lst) · one-item tuple needs the comma (1,) · string methods return NEW strings: s = s.upper() · zip silently stops at the shortest — strict=True · never shadow built-ins (list, dict, str, type, id as variable names).

9 · Teardown — getting out

leave REPL · ctrl-c · pip uninstall · del · close files
exit() / quit() / ctrl dleave the REPL — Windows: ctrl z then Enter. Python 3.13+ accepts bare exit too
ctrl cstop a running script — raises KeyboardInterrupt, a real exception you can catch (except KeyboardInterrupt:) for graceful shutdown
pip uninstall requestsremove a package from the env's site-packages; its dependencies stay
del xremoves the name binding; the object is freed by the garbage collector once nothing references it — del ≠ destroy
f.close()…or skip it: with open(...) as f: calls close() on exit even if an exception fires — the whole point of context managers (§6)
