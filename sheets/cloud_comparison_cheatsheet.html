0 · The mental model (read once)
AWS = Amazon Web Services (2006, first mover, biggest catalog, quirkiest names). Azure = Microsoft Azure (2010, enterprise/hybrid play, glued to Windows/Office/Entra identity). GCP = Google Cloud Platform (2008, infrastructure-first from running Google itself; strongest in data/analytics/ML).
All three sell the same ~20 primitives — VMs, buckets, managed DBs, a software-defined network, identity, billing hierarchy. Knowing one cloud, learning another is ~80% vocabulary translation, ~20% real conceptual differences. The 20% lives in (1) the resource/billing hierarchy (section 1 — actually think about this one) and (2) scope defaults — e.g. a GCP VPC is global, an AWS VPC / Azure VNet is regional. Same word, different blast radius.

Rule of thumb: forget a GCP name → guess Cloud <noun>, often right. Forget an AWS name → you can't guess, look it up. Azure sits in between.

1 · IAM & org hierarchy — the conceptual heart

IAM = Identity and Access Management: who (identity) can do what (role) on which resource (scope). All three inherit permissions down the hierarchy — what differs is what the layers are.

AWS                      Azure                         GCP
───                      ─────                         ───
Organization             Tenant (Entra ID directory)   Organization
 └─ Organizational        └─ Management Group           └─ Folder
    Unit (OU)                 └─ Subscription               └─ Project
     └─ Account                   └─ Resource Group             └─ Resources
         └─ Resources                 └─ Resources

Layer-by-layer

AWSAzureGCPRole the layer plays
AWS Organizations (mgmt account)Tenant — the Entra ID directory itselfOrganizationidentity directory + root of trust
Organizational Unit (OU) — nestableManagement Group — nestable, up to 6 levelsFolder — nestable (dept / team / env)grouping layer for policy inheritance

The rough equation: AWS Account ≈ Azure Subscription ≈ GCP Project — each is the unit of isolation, quota, and billing attribution. But the weight differs: an AWS account is heavy (companies script account factories); an Azure subscription is medium; a GCP project is deliberately cheap — one per app per environment is normal.
Azure's two oddities: (1) the Resource Group has no AWS/GCP container equivalent — a mandatory folder inside a subscription, deletable as a unit (GCP achieves the same by deleting a project). (2) Azure splits identity from resources: the Tenant (Entra ID, ex-Azure Active Directory) exists even with zero subscriptions and is shared with Microsoft 365.
Billing attaches differently: AWS per account (consolidated at org) · Azure per subscription · GCP a separate billing account object linked to projects — one billing account can pay for many projects. ACE exam loves this.
