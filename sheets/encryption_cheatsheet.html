Encryption & Password Entropy Cheat Sheet

Entropy · cracking timelines · quantum reality check · started 2026-06-06 · twin of Tools/encryption_cheatsheet.md

Scenario assumed throughout: the attacker already holds the encrypted vault / hashed password file and brute-forces offline — the only case where password strength is the primary defense (online logins are rate-limited).

0 · The mental model: entropy = doublings

Entropy measures unpredictability, in bits; each extra bit doubles the guesses required. 40 bits ≈ 1 trillion possibilities; 80 bits ≈ 1.2 septillion. Formula: entropy = log₂(possible passwords) — random chars: log₂(alphabet) × length; random words: log₂(wordlist) × words (a 7,776-word diceware list = 12.9 bits/word).
Etymology: "entropy" comes from thermodynamics (Clausius, 1865 — Greek entropia, "transformation"); Claude Shannon imported it into information theory in 1948, allegedly on von Neumann's advice that "no one knows what entropy really is, so in a debate you will always have the advantage."

1 · Entropy of common password approaches

Entropy depends on how it was generated, not how it looks. Complex-looking human choices are weak; simple-looking random words are strong.

🎲 The key word is RANDOM. These numbers hold only under uniform random selection — dice, a CSPRNG (Cryptographically Secure Pseudo-Random Number Generator), or a password manager's generator. Words you pick cluster around familiar, emotionally salient choices and lose most of their entropy: 12.9 bits/word holds only for true random draws. Use a passphrase generator or physical dice + a published wordlist.

2 · How fast can attackers guess?

The hash algorithm sets the speed limit. Fast hashes are catastrophic for passwords; slow, memory-hard hashes are the defense.

AlgorithmGuesses/sec (1× RTX 5090, 2025)12-GPU rigDesign intent
MD5 (obsolete)~160 billion~1.9 trillionnever meant for passwords
SHA-256~22 billion~260 billiongeneral-purpose hash, not for passwords
bcrypt (cost 10)~10,000~120,000deliberately slow; common in web frameworks
Argon2id (64 MB, 3 iter)~1,000–3,000~12,000–36,000memory-hard: 64 MB per guess; resists GPU parallelism
Real-world anchor: Bitwarden's default Argon2id (64 MB, 3 iterations, parallelism 4) holds a stolen-vault attacker with a top GPU rig to ~20k guesses/sec — not billions. Argon2id won the 2015 Password Hashing Competition; "memory-hard" = every guess must occupy real RAM, which GPU farms can't fake cheaply.

4 · Time to crack — quantum computing

Grover's algorithm gives brute-force search a quadratic speedup — in practice it halves effective entropy: a 64-bit password resists a quantum attacker like a 32-bit one resists a classical attacker. (Shor's algorithm, the other quantum threat, breaks RSA/ECC keys — different target than passwords.)
⚠️ Timeline reality check (early 2026): no quantum computer can run Grover's against real passwords. Current machines: <2,000 noisy physical qubits; an attack needs thousands of logical (error-corrected) qubits, each built from thousands of physical ones. The National Academies (2019) put AES-128 via Grover's at >10¹² years on then-projected hardware. Not expected within a decade.

CRQC = Cryptographically Relevant Quantum Computer — powerful enough to run Grover's/Shor's against real targets; none exists. "~2036" assumes ~10⁶ Grover iterations/sec (speculative); "2100" assumes ~10¹² (aggressive upper bound). The defense is plain: longer passwords — entropy that survives halving.

✅ Practical takeaway: a 4-word random master passphrase is safe against everything that exists today or this decade; use 5–6 words for margin against quantum machines that may emerge in your lifetime. Generated 20+-char passwords in a manager are already quantum-resistant. And the real threat is none of this — it's phishing, credential reuse, and social engineering, which passkeys eliminate. Migrating to passkeys matters more than adding a fifth word.

6 · Harvest Now, Decrypt Later (HNDL)

HNDL = adversaries record encrypted traffic today to decrypt later with quantum computers. Real for data in transit (TLS sessions, VPN tunnels, encrypted email) — hence NIST's 2024 post-quantum standards: ML-KEM (key encapsulation, from CRYSTALS-Kyber) and ML-DSA (signatures, from CRYSTALS-Dilithium).
HNDL does not apply to a password-manager master password: it never crosses the network — it's processed locally through Argon2id to derive the encryption key. Same for FileVault passphrases and device PINs. For vaults, the threat model remains "server breach exposes encrypted file" → the brute-force timelines above.
