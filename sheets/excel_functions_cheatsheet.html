Excel Functions Cheat Sheet

The favourites: LAMBDA · CUBEVALUE · DATEDIF · EOMONTH · started 2026-06-06 · twin of Tools/excel_functions_cheatsheet.md

0 · The mental model & formula grammar (read once)

Excel function = a named, built-in formula that takes arguments (inputs, separated by commas) and returns a value into the cell where it's written. The grammar is always =NAME(arg1, arg2, …) — the leading = tells Excel "this is a formula, not text."

The four headliners share a theme: each one replaces something clunkier — LAMBDA replaces VBA macros for custom math, CUBEVALUE replaces fragile GETPIVOTDATA dashboards, DATEDIF replaces hand-rolled date subtraction, and EOMONTH replaces error-prone month-end arithmetic. (Scope: Windows Excel, Microsoft 365.)
Four functions, four jobs: 
  LAMBDA      →  "teach Excel a new verb"        (custom reusable functions)
  CUBEVALUE   →  "ask the Data Model directly"   (dashboard cells, no pivot table)
  DATEDIF     →  "how long between two dates?"   (tenure, durations — hidden but stable)
  EOMONTH     →  "snap to a month boundary"      (close dates, billing periods)
Memory hook: Learn, Consult, Duration, Deadline — the first two create and query, the last two do calendar math.
Formula grammar: =FUNCTION(required, [optional], …)
  • = — starts a formula.
  • FUNCTION — case-insensitive; Excel capitalizes it for you.
  • [square brackets] in documentation mean optional argument — you never type the brackets themselves.
  • Arguments are separated by , (or ; on systems with European regional settings — France included; worth knowing when sharing files across regions).
  • Cell reference = an address like B2; range = a rectangle like B2:B13; structured reference = a table-column name like Wells[Spud_Date] (readable, auto-expanding — prefer these).

1 · LAMBDA — reusable custom functions, no VBA

LAMBDA lets you define your own function in pure formula language and call it by name anywhere in the workbook. Named after lambda calculus (Alonzo Church, 1930s), the mathematical notation for anonymous functions that underlies functional programming. Before LAMBDA (added to Microsoft 365 in 2021), custom functions required VBA (Visual Basic for Applications — Excel's embedded macro language), which means macro-enabled .xlsm files, security warnings, and code most colleagues can't read.

Syntax grammar

=LAMBDA(parameter1, [parameter2, …], calculation)
        └────────── names you invent ──────────┘ └─ formula using them ─┘

Up to 253 parameters; the last argument is always the calculation. A bare LAMBDA in a cell returns #CALC! — it's a function definition, not a result. To test it inline, append arguments in a second parenthesis pair: =LAMBDA(x, x^2)(4) → 16.

The define-in-Name-Manager workflow (this is the whole trick)

1. Formulas ▸ Name Manager ▸ New…shortcut Ctrl+F3
2. Name:what you'll type in formulas, e.g. WACC
3. Refers to:the full LAMBDA, starting with =
4. OKnow =WACC(…) works in any cell of that workbook, with IntelliSense

Worked examples

Custom WACC (Weighted Average Cost of Capital — the blended cost of a project's debt + equity financing, the discount rate in NPV screening):

Name:      WACC
Refers to: =LAMBDA(equity, debt, cost_e, cost_d, tax,
              (equity*cost_e + debt*cost_d*(1-tax)) / (equity+debt))

Usage:     =WACC(700, 300, 0.12, 0.06, 0.25)   → 0.0975  (9.75%)

Unit conversion — barrels to cubic metres (1 bbl = 0.158987 m³):

Name:      BBL2M3
Refers to: =LAMBDA(bbl, bbl * 0.158987)
Usage:     =BBL2M3(50000)   → 7949.35

Pair with LET for readability. LET assigns names to intermediate values inside a formula — =LET(name1, value1, …, calculation) — so a complex LAMBDA reads like a small program instead of a one-liner:

Name:      DECLINE        (exponential production decline: rate after t years)
Refers to: =LAMBDA(q0, d, t,
              LET(decay, EXP(-d*t),
                  q0 * decay))
Usage:     =DECLINE(1200, 0.15, 3)   → 765.2 bbl/d

Helper functions that take a LAMBDA as input

MAP(array, LAMBDA(x, …))Apply the LAMBDA to every cell of an array, return same-shaped array.
BYROW(array, LAMBDA(row, …))One result per row — e.g. =BYROW(B2:E13, LAMBDA(r, MAX(r))) = monthly peak per well.
BYCOL(array, LAMBDA(col, …))Same, per column.
SCAN(init, array, LAMBDA(acc, x, …))Running accumulation — e.g. cumulative production: =SCAN(0, B2:B13, LAMBDA(a, v, a+v)).
REDUCE(init, array, LAMBDA(acc, x, …))Like SCAN but returns only the final value.
Gotchas. Names live per workbook — copy a sheet to a new workbook and the LAMBDA names come along, but a fresh workbook starts empty (share via a template or the Advanced Formula Environment add-in). No self-documentation of parameter order — name parameters descriptively (cost_of_equity, not x). Recursion is allowed (a LAMBDA can call itself by its defined name) but easy to crash — there's a recursion depth limit.

Related: LETMAPBYROWBYCOLSCANREDUCEMAKEARRAYISOMITTED

2 · CUBEVALUE — query the Data Model directly

Cube functions (the seven CUBE… functions) pull single values straight out of an OLAP cube — including the workbook's own Data Model (the in-memory Power Pivot engine inside Excel) — without needing a pivot table on the sheet.

OLAP (Online Analytical Processing) = a database organized for slice-and-dice analysis: numeric measures (e.g. Total CAPEX) viewed across dimensions (e.g. Project, Month, Cost Category).
MDX (MultiDimensional eXpressions) = the query language for OLAP cubes. Cube-function member strings like "[Project].[Name].[Alpha]" are MDX paths: [Dimension].[Hierarchy].[Member].
Member = one item in a dimension ("Project Alpha", "2026-05"); measure = the number being aggregated ([Measures].[Total Spend]). 
=CUBEVALUE(connection, member_expression1, [member_expression2, …])
           └─ usually "ThisWorkbookDataModel" ─┘ └─ filters that intersect ─┘

Each member expression narrows the slice; the result is the measure value at their intersection — exactly one cell of an imaginary pivot table.

KPI cell — May 2026 spend on one project, no pivot table needed:

=CUBEVALUE("ThisWorkbookDataModel",
           "[Measures].[Total Spend]",
           "[Projects].[Name].[Alpha Compression]",
           "[Calendar].[Month].[2026-05]")

Better layout — CUBEMEMBER in helper cells. CUBEMEMBER fetches one member as a reusable handle; point CUBEVALUE at those cells so a whole dashboard re-targets by editing one cell:

B1: =CUBEMEMBER("ThisWorkbookDataModel","[Projects].[Name].[Alpha Compression]")
B2: =CUBEMEMBER("ThisWorkbookDataModel","[Calendar].[Month].[2026-05]")
B4: =CUBEVALUE("ThisWorkbookDataModel","[Measures].[Total Spend]", B1, B2)
B5: =CUBEVALUE("ThisWorkbookDataModel","[Measures].[Budget]",      B1, B2)
B6: =B4/B5            ← spend vs budget KPI, plain arithmetic on cube results

Slicer connection trick. A slicer (the clickable filter buttons) can drive CUBEVALUE directly — pass the slicer's name as an argument:

=CUBEVALUE("ThisWorkbookDataModel","[Measures].[Total Spend]", Slicer_Project)

Find the slicer's formula name under Slicer Settings ▸ Name to use in formulas. Now clicking the slicer re-filters your hand-built dashboard with no pivot table anywhere.

Why it beats GETPIVOTDATA for dashboards. GETPIVOTDATA reads values off an existing pivot table, so the pivot must stay on a sheet and keep its layout — collapse a field or refresh differently and your references break. CUBEVALUE talks to the Data Model itself: cells go anywhere, formatted however you like, immune to pivot-layout changes. Memory hook: GETPIVOTDATA scrapes the report; CUBEVALUE asks the database.
Gotchas. Requires data loaded to the Data Model (Power Pivot, or Power Query with "Add to Data Model" checked) — plain ranges/tables won't answer. #N/A while connecting is normal for a moment; persistent #N/A usually means a typo in the MDX path — type member strings slowly, Excel autocompletes the path segments inside the quotes. Misspelled measure/member names fail silently to #N/A with no hint where. Many CUBEVALUEs recalculate per refresh; thousands of them get slow — batch via CUBEMEMBER handles.

Related: CUBEMEMBERCUBESET (define a set, e.g. "top 5 wells") CUBESETCOUNTCUBERANKEDMEMBER (nth item of a set) CUBEMEMBERPROPERTYCUBEKPIMEMBER

3 · DATEDIF — the famous "hidden" date-difference function

DATEDIF returns the difference between two dates in whole years, months, or days. It's Excel's most famous ghost: inherited from Lotus 1-2-3 (the dominant 1980s spreadsheet Excel had to stay compatible with), it works in every Excel version but Microsoft barely documents it — no autocomplete, no tooltip, no entry in the function wizard. Undocumented but stable: it's been there ~30 years and isn't going anywhere.

=DATEDIF(start_date, end_date, unit)
         └─ earlier ─┘ └─ later ─┘ └─ a text code, in quotes ─┘

start_date must be ≤ end_date, or you get #NUM! (it never returns negatives — swap the arguments instead).

Unit codes

UnitReturnsExample (2024-03-15 → 2026-06-06)
"Y"Complete years2
"M"Complete months26
"D"Days (same as plain subtraction)813
"YM"Months, ignoring years — the remainder after "Y"2
"MD"Days, ignoring months & years — ⚠ buggy, avoid22 (when it works)
"YD"Days, ignoring years83
⚠ The "MD" bug — Microsoft's own documentation warns that "MD" can return wrong (even negative) results around month boundaries, because of how it borrows days from months of different lengths. Don't use it. For the "leftover days" part of a duration, compute it yourself: end_date - EDATE(start_date, DATEDIF(start_date, end_date, "M")).

Service tenure in words (employee start date in B2):

=DATEDIF(B2, TODAY(), "Y") & " yr " & DATEDIF(B2, TODAY(), "YM") & " mo"
→ "7 yr 4 mo"

Project phase duration in whole months (FID date in B2, first-oil forecast in C2 — FID = Final Investment Decision):

=DATEDIF(B2, C2, "M")     → 26    (complete months between sanction and first oil)

Contract days elapsed / remaining (award B2, completion C2):

Elapsed:   =DATEDIF(B2, TODAY(), "D")
Remaining: =C2 - TODAY()             ← plain subtraction is fine for days
Gotchas. You must type it blind — no autocomplete or argument tooltip appears; that's normal, not a sign it's broken. "Complete" units truncate, never round: 1 year 11 months → "Y" returns 1. #NUM! = your start date is after your end date. For working-day counts use NETWORKDAYS(start, end, [holidays]) instead — DATEDIF counts calendar days.

Related: YEARFRAC (fractional years — good for interest/depreciation) NETWORKDAYSNETWORKDAYS.INTL (business days) EDATETODAY

4 · EOMONTH — month-end arithmetic

EOMONTH (End Of MONTH) returns the serial number of the last day of the month that is N months from a start date. Excel stores dates as serial numbers (days elapsed since 1900-01-01), so "last day of the month" is genuinely fiddly to compute by hand — months have 28/29/30/31 days — and EOMONTH does it in one call.

=EOMONTH(start_date, months)
         └─ anchor ─┘ └─ offset: 0 = this month, 1 = next, -1 = previous ─┘

Format the result cell as a date — otherwise you'll see the raw serial number.

Month-end close date for the month containing any transaction date B2:

=EOMONTH(B2, 0)          → 2026-06-30   (for any B2 in June 2026)

Invoice due date — end of next month (common "net EOM+1" payment terms):

=EOMONTH(B2, 1)          → 2026-07-31

Quarter-end from any date (jump to the month that closes the quarter):

=EOMONTH(B2, 2 - MOD(MONTH(B2)-1, 3))
The pattern worth memorizing: first of next month. 
=EOMONTH(date, 0) + 1    → first day of the NEXT month
=EOMONTH(date,-1) + 1    → first day of the CURRENT month
Memory hook: the day after month-end is always the 1st. This pair generates clean period boundaries for accrual schedules, depreciation start dates, and monthly bucketing — e.g. a 12-row schedule of period starts: 
=EOMONTH(start, SEQUENCE(12,1,0)) + 1

EDATE — the sibling

EDATE shifts a date by N months but keeps the same day-of-month (clamping when the day doesn't exist: Jan 31 + 1 month → Feb 28/29):

=EDATE("2026-01-15", 6)   → 2026-07-15    (6-month review date)
=EDATE(spud_date, 18)     → 18-month milestone, same day-of-month

Rule of thumb: EDATE preserves the day; EOMONTH snaps to the end.

Gotchas. Unformatted result shows a number like 46203 — that's the date serial, not an error; apply a date format. Negative months goes backward — handy, easy to forget it's allowed. Text dates like "2026-06-06" usually coerce fine, but real date serials (or DATE(2026,6,6)) are safer across regional settings. For "last working day of the month": =WORKDAY(EOMONTH(B2,0)+1, -1).

Related: EDATEWORKDAYWORKDAY.INTLDATEDAYMONTHSEQUENCE

5 · Natural extensions — six modern functions to adopt next

LET(n1, v1, …, calc)Name intermediate values inside a formula — long formulas become readable and compute each step once.
XLOOKUP(find, in, return)The modern lookup: replaces VLOOKUP/HLOOKUP/INDEX-MATCH, looks any direction, exact-match by default, has an if-not-found argument.
FILTER(array, condition)Pull all rows matching a condition into a live spilled range — e.g. all open AFEs (Authorization For Expenditure) over budget.
SEQUENCE(rows, [cols], [start], [step])Generate number/date series in one formula — period numbers, schedule rows, loop indexes for LAMBDA helpers.
TEXTSPLIT(text, delimiter)Split "WELL-A12/Zone3" style codes into columns by delimiter — Text-to-Columns as a live formula.
SUMIFS(sum_range, criteria_range1, criteria1, …)Conditional totals across multiple criteria — spend by project AND month AND cost code in one call.

These all spill: a formula returning an array fills neighbouring cells automatically (blue border); #SPILL! means something is blocking the landing zone.

When Excel → Python/pandas is the better tool. Excel wins for interactive what-if work, shared dashboards, and anything a colleague must open and audit. Switch to pandas (Python's table-manipulation library) when: data exceeds ~500k rows or Excel slows to a crawl; the same clean-merge-pivot pipeline runs every week (script it once, rerun forever); you need joins across many files; logic needs version control and testing (Git diffs work on .py, not .xlsx); or the output feeds another system rather than a human. Rule of thumb: Excel for thinking and showing, pandas for repeating and scaling. A LAMBDA library is often the last comfortable stop before that switch.

6 · Teardown — cleaning a workbook before it leaves you

Remove a LAMBDAFormulas ▸ Name Manager (Ctrl+F3) ▸ select ▸ Delete — any formula still calling it turns #NAME?, so find-and-replace usages first
Freeze formulas to static valuesselect ▸ copy ▸ Paste Special ▸ Values — locks results in place so recipients see numbers, not formulas that may break on their machine
Break external workbook linksData ▸ Edit Links ▸ Break Link — referenced values become static; no more "this workbook contains links" prompts
Disconnect CUBEVALUE from its sourceData ▸ Queries & Connections ▸ delete the connection — but value-freeze the CUBE cells first, or they all go #N/A
Why bother before emailing?recipients lack your Data Model, file paths, and named LAMBDAs — without teardown they open a file full of update prompts, #NAME? and #N/A
