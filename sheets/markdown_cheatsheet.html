Markdown Cheat Sheet

CommonMark · GFM · Obsidian · started 2026-06-12 · twin of Tools/markdown_cheatsheet.md

0 · The mental model (read once)

Markdown = a lightweight markup language by John Gruber with Aaron Swartz, 2004. You write plain text with a few punctuation conventions; a renderer converts it to HTML (HyperText Markup Language, what browsers display). The goal: source as readable as the output. The name is a pun on "markup" — markup that gets out of your way.
Readable plain text → formatted output. You never see tags while writing: 
 you write           renderer           reader sees
(.md plain text)  →  (Markdown →    →   (formatted HTML
                      HTML engine)       in a browser)

# Hello           →   <h1>Hello</h1> →   Hello  (big heading)
**bold**          →   <b>bold</b>    →   bold
- milk            →   <li>milk</li>  →   • milk
Source = a .md plain-text file. Renderer = the converter (GitHub, Obsidian, a site builder) — this is where dialects diverge. Output = usually HTML. Why it won: plain text outlives apps and reads cleanly even unrendered.
Three dialects: CommonMark = the standard baseline GFM = + tables, tasks, strikethrough Obsidian = + wikilinks, callouts, frontmatter

1 · The three dialects you'll meet

CommonMarkthe 2014 standard spec that pinned down Markdown's ambiguities — the baseline everyone agrees on (Reddit, Stack Overflow, most libraries)
GFM (GitHub-Flavored Markdown)CommonMark + tables, task lists, strikethrough, autolinks, syntax highlighting (GitHub/GitLab READMEs, issues, PRs)
ObsidianCommonMark/GFM + [[wikilinks]], ![[embeds]], callouts, #tags, YAML frontmatter (the Obsidian app)

Sections 2–3 are core (everywhere). 4 is GFM. 5 is Obsidian.

2 · Core syntax (CommonMark — works everywhere)

Text & inline

# H1 … ###### H6headings, levels 1–6 — space after the #
*italic* / _italic_one marker each side
**bold**two markers
***bold italic***three markers
`code`inline code (commands, filenames, identifiers)
[text](https://url)link — text in [], URL in ()
![alt](img.png)image — like a link with a leading !
\*literal\*backslash escapes a special char (§7)

Blocks

> quotedblockquote; >> nests
--- / ***horizontal rule, own line, blank line above
- item / * itemunordered list (bullet)
1. itemordered list (numbers can all be 1.)
indent 2–4 spacesnests a sub-list under its parent

Reference link — define once, reuse: [cm]: https://spec.commonmark.org then [text][cm]. Keeps prose readable for long URLs.

CommonMark gotcha — blank line before a list. A list needs a blank line separating it from the paragraph above, or strict renderers glue it into the preceding text.

Fenced code blocks

Three backticks; add a language hint after the opening fence for syntax highlighting.

```python
def greet(name):
    return f"hello {name}"
```

Line breaks — the #1 gotcha

A single newline does not break the line — the renderer collapses it to a space.

new paragraphleave a blank line
break within a paragraphend the line with two trailing spaces

The two-trailing-spaces rule is invisible and infamous. (GFM/Obsidian also accept a line-end \ or honour single newlines — §6.)

3 · Inline HTML is valid Markdown

Drop raw HTML straight into Markdown — most renderers pass it through. The escape hatch for anything Markdown can't express (<kbd> keys, collapsible blocks, fine layout). 
Here is a <kbd>Ctrl</kbd>+<kbd>C</kbd> hint and a <br> break.

<details>
<summary>Click to expand</summary>
Hidden content here.
</details>

Caveat: some renderers sanitize or strip HTML for safety (GitHub strips <script>) — see §6. Cross-ref html_cheatsheet.md.

4 · Tables & GitHub-Flavored Markdown (GFM)

Tables (GFM, not core)

Pipes | split columns; a dash row splits header from body. Colons set alignment.

| Left | Center | Right |
|:-----|:------:|------:|
| a    | b      | c     |
:---left (default)
:--:center
---:right

Source spacing needn't line up — the renderer parses by position, not your whitespace.

GFM extras

- [ ] / - [x]task list — rendered (and on GitHub, clickable) checkboxes
~~gone~~strikethrough
https://... (bare)autolink — no []() needed
` ```js ` fencesyntax-highlighted code block
text[^1] … [^1]: notefootnote — superscript ref + note at the bottom

Strikethrough is GFM, not core — ~~text~~ does nothing in strict CommonMark.

5 · Obsidian extras (app-specific)

None of these render on plain GitHub — they show as literal text there. See obsidian_cheatsheet.md.

Links, embeds, tags

[[Note Name]]wikilink — link a note by its title
[[Note#Heading]]jump to a heading in that note
[[Note#^id]]link to one specific block
[[Note|text]]aliased — link shows custom text
![[Note]] / ![[img.png]]embed — inline the content of a note/image
#project #area/financetag — searchable label; / nests

Callouts & YAML frontmatter

A callout is a styled blockquote; [!type] picks the style.

> [!note] Optional title
> Body text of the callout.

Types: note, tip, info, warning, danger, quote, example.

Frontmatter = a YAML (YAML Ain't Markup Language) key: value block at the very top, fenced by ---. Sets title/tags/date.

---
title: Project Plan
tags: [project, q3]
created: 2026-06-12
---

6 · Renders differently everywhere

The same Markdown is not guaranteed to look the same in two places. Markdown was under-specified for a decade; CommonMark fixed the core, but dialects diverge on the edges. Test where your readers will actually see it.
Line breaksstrict CommonMark needs two trailing spaces; GitHub comments, Obsidian, chat apps honour a single newline
Nested listsindent width (2 vs 4 spaces) and whether sub-lists "take" varies
Raw HTMLallowed (GitHub, static sites), partly sanitized (GitHub strips <script>), or stripped (some chat fields)
Tables / strikethrough / task listsGFM-only — invisible in strict CommonMark
Wikilinks / embeds / calloutsObsidian-only — show as literal [[...]] on GitHub

7 · Gotchas & escaping

Common bites

literal * _ # ` [ eaten as formattingprefix with backslash: \*not italic\*
line break not appearingsingle newline used — add two trailing spaces or a blank line
list glued onto the paragraph aboveinsert a blank line before the list
backticks inside inline codefence with more backticks

More bites

#tag vs #heading# + space = heading; # glued to a word = (Obsidian) tag
Markdown ignored in a code blockby design — fenced/indented code is verbatim
HTML you wrote shows as textthe renderer sanitized it (§6)

Rule of thumb: punctuation being interpreted but you wanted it literal → put a \ in front. The backslash only shows when it precedes a Markdown-special character.

8 · Glossary recap

Markdownlightweight markup language (Gruber + Swartz, 2004); readable plain text → HTML. Name = pun on "markup"
Markup languageplain-text annotations telling software how to structure/format a document (HTML, XML, LaTeX)
CommonMarkthe 2014 standard spec disambiguating Markdown; the baseline dialect
GFMGitHub-Flavored Markdown; CommonMark + tables, task lists, strikethrough, autolinks, highlighting. Anchor: "GitHub's extras"
HTMLHyperText Markup Language; what browsers render & what Markdown compiles to
YAMLYAML Ain't Markup Language (self-referential); human-readable key: value data used for frontmatter
Frontmattera YAML metadata block at the top of a .md file, fenced by --- (title, tags, date)
Fencea code block delimited by ``` lines, optionally tagged with a language
WikilinkObsidian's [[Note]] linking by title (vs Markdown's [text](url))
CalloutObsidian's > [!type] styled blockquote (admonition box)
Rendererthe engine converting Markdown → output; where dialects diverge
